## Project boundary decision

**Don't Terraform the project itself.** Precedent from AKS: we manage the *resource group* (per-deployment boundary) but not the *subscription* (commercial billing boundary). GCP has no resource group â€” the project IS the boundary. Applying the same principle: the project is fixed click-ops infrastructure; Terraform manages resources *inside* it.

Trade-offs considered:

| For TF-managing the project | Against |
|---|---|
| Audit trail on billing-account swaps / label changes | Importing console-created project causes label/metadata drift every run |
| Reproducibility if rebuilt in a different org | `tofu destroy` blast radius includes project (30-day GCP grace mitigates but zero upside) |
| Everything-as-code purity | Bootstrap chicken-and-egg: org-level permissions required |

If tracking the billing-account association becomes valuable later, add a read-only `data "google_billing_account"` block â€” no lifecycle risk.

### `variables.tf`

| Variable | Type | Default | Notes |
|---|---|---|---|
| `project_id` | string | (required) | Injected via `TF_VAR_project_id` from `.env.local`. Value: `trakrf-494211`. |
| `region` | string | `"us-central1"` | Iowa. Cheapest standard region with ARM `t2a-standard-*` SKU availability for phase-2 cluster parity. |
| `zone` | string | `"us-central1-a"` | Phase-2 single-zone pin for CNPG PV affinity (TRA-364 pattern). Declared here so the variable set is complete from the start. |
| `environment` | string | `"demo"` | Matches `aws/`, `azure/`. |
| `project` | string | `"trakrf"` | Matches `aws/`, `azure/`. (GCP "project" is a separate concept â€” this is the label value.) |
| `location` | string | `"usc1"` | Short code for resource naming. `usc1` = us-central1. Peer of azure's `ussc`. |
| `cluster_name` | string | `"trakrf-demo"` | Matches EKS/AKS demo cluster_name. Applied in TRA-460. |

### `main.tf`

```hcl
locals {
  region      = var.region
  name_prefix = "${var.project}-${var.environment}-${var.location}"  # trakrf-demo-usc1

  common_labels = {
    project     = var.project
    environment = var.environment
    managed_by  = "terraform"
    ticket      = "tra-459"
  }
}

resource "random_string" "suffix" {
  length  = 4
  special = false
  upper   = false
}
```

GCP resource labels must be lowercase alphanumeric + `_`/`-`, so `common_labels` keys/values differ in case from Azure's `common_tags` (which were title-case). Ticket tag is lowercase for the same reason.

### `provider.tf`

```hcl
terraform {
  required_version = ">= 1.0.0"

  required_providers {
    google = { source = "hashicorp/google", version = "~> 6.0" }
    random = { source = "hashicorp/random", version = "~> 3.6" }
  }

  backend "s3" {
    bucket  = "tf-state"
    key     = "gcp.tfstate"
    region  = "auto"
    profile = "cloudflare-r2"

    skip_credentials_validation = true
    skip_region_validation      = true
    skip_metadata_api_check     = true
    use_path_style              = true
  }
}

provider "google" {
  project = var.project_id
  region  = var.region
  zone    = var.zone
}
```

Auth: the `google` provider resolves credentials via the Application Default Credentials chain (`GOOGLE_APPLICATION_CREDENTIALS` env â†’ `gcloud auth application-default login` fallback). No key material committed.

### `apis.tf`

```hcl
# TRA-459: required service APIs for the GKE rollout.
# Out of scope for phase 1: artifact registry, workload identity, IAM credentials
# APIs â€” those land in TRA-460/phase 4 alongside the resources that need them.

resource "google_project_service" "required" {
  for_each = toset([
    "container.googleapis.com",           # GKE
    "compute.googleapis.com",             # hard dep of GKE (VMs, networks, disks)
    "dns.googleapis.com",                 # Cloud DNS (phase 2)
    "iam.googleapis.com",                 # IAM
    "cloudresourcemanager.googleapis.com" # project metadata / IAM bindings
  ])

  service = each.key

  # Keep services enabled even if this resource is removed from state.
  # Disabling APIs on a live project can break running resources.
  disable_on_destroy = false
}
```

`compute.googleapis.com` is added beyond the four APIs listed in the ticket because GKE transitively requires it (phase-2 `tofu apply` fails without it). Listing it explicitly at scaffold time avoids a surprise in phase 2.

## Justfile integration

Append after the existing `azure` recipe:

```make
# Plan and apply GCP infrastructure (GKE, Cloud DNS, Artifact Registry)
gcp: (_backend-conf "terraform/gcp")
    @echo "Planning GCP infrastructure..."
    @tofu -chdir=terraform/gcp init -backend-config=backend.conf
    @tofu -chdir=terraform/gcp plan -out=tfplan
    @tofu -chdir=terraform/gcp apply tfplan
```

No new `export TF_VAR_*` lines needed at the top of the file â€” `TF_VAR_project_id` is set in `.env.local` under its natural name and picked up by Terraform via direnv.

## CI integration

Single-line matrix addition to `.github/workflows/ci.yml`:

```yaml
strategy:
  fail-fast: false
  matrix:
    dir: [aws, azure, cloudflare, bootstrap, gcp]
```

`tofu validate` runs with `-backend=false` so no GCP credentials required in CI. Provider schema downloads during `init`, HCL syntax and schema validate, exits clean. `tofu fmt -check -recursive terraform/` already covers the new module.
