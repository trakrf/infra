# TRA-461 GKE Phase 3 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Get TrakRF running on the TRA-460 GKE cluster end-to-end: Workload-Identity-backed cert-manager minting a real Let's Encrypt wildcard cert for `gke.trakrf.app`, Traefik on a static regional IP, CNPG + trakrf-backend + trakrf-ingester deployed via the existing ArgoCD root-app, and a manual smoke test (login → BLE scan → inventory save) confirming the full data path.

**Architecture:** Additive port of TRA-438 (AKS phase 3). Terraform adds a GCP service account federated to the `cert-manager` K8s SA with `roles/dns.admin` scoped to the `gke.trakrf.app` Cloud DNS zone only, a static regional `google_compute_address` for Traefik's LB, and TF-managed apex + wildcard A records. Each `helm/*` chart gets a `values-gke.yaml` overlay alongside existing `values-aks.yaml`/`values-eks.yaml`. The `argocd/root/` Helm chart grows a `gke` branch in three conditionals (cert-manager-config, cert-manager, traefik) and three new value keys. `scripts/apply-root-app.sh` gets a `gke)` case. No refactor of the AKS pattern; provider-neutralization defers to cluster #3.

**Tech Stack:** OpenTofu + Google provider v6, Helm 3, ArgoCD 2.x, cert-manager + Cloud DNS solver via GKE Workload Identity, `gcloud`, `gke-gcloud-auth-plugin`, `just`, `kubectl`.

**Spec:** `docs/superpowers/specs/2026-04-23-tra-461-gke-phase-3-design.md`

**Branch:** `feature/tra-461-gke-phase-3` (already created; spec committed at `5f544cd`)

### Task 2: Create cert-manager GCP service account + zone-scoped DNS role + Workload Identity binding

Provisions the GCP-side identity cert-manager federates into via GKE Workload Identity. Scoped tightly: `roles/dns.admin` on the `gke.trakrf.app` managed zone only, not project-wide.

**Files:** `terraform/gcp/cert_manager.tf` (create)

- [ ] **Step 1: Write `cert_manager.tf`**

Create `terraform/gcp/cert_manager.tf`:

```hcl
# GCP service account that cert-manager pods federate into via GKE Workload Identity.
# Scoped to roles/dns.admin on the gke.trakrf.app zone ONLY — the Cloud DNS solver
# needs TXT record create/delete for _acme-challenge.* during DNS-01 validation.
resource "google_service_account" "cert_manager" {
  account_id   = "cert-manager-${var.environment}"
  display_name = "cert-manager DNS-01 solver (${local.name_prefix})"
  description  = "Used by cert-manager pods via Workload Identity to solve ACME DNS-01 on gke.trakrf.app"
}

# Zone-scoped role binding: grants dns.admin ONLY on the gke-trakrf-app managed zone.
# Uses google_dns_managed_zone_iam_member rather than a project-level google_project_iam_member
# so the blast radius stays tight — cert-manager cannot touch other zones or project resources.
resource "google_dns_managed_zone_iam_member" "cert_manager_dns_admin" {
  project      = var.project_id
  managed_zone = google_dns_managed_zone.gke_trakrf_app.name
  role         = "roles/dns.admin"
  member       = "serviceAccount:${google_service_account.cert_manager.email}"
}

# Workload Identity binding: grants the Kubernetes SA cert-manager/cert-manager
# the ability to impersonate this GCP SA. Subject must exactly match the actual
# Helm-chart-default cert-manager SA namespace/name.
resource "google_service_account_iam_member" "cert_manager_wi" {
  service_account_id = google_service_account.cert_manager.name
  role               = "roles/iam.workloadIdentityUser"
  member             = "serviceAccount:${var.project_id}.svc.id.goog[cert-manager/cert-manager]"
}
```

### Task 3: Create Traefik static regional IP

IP stability across cluster rebuilds. `prevent_destroy` guards against accidental rotation — TF-managed A records in `dns.tf` depend on this IP being stable. GCE cloud-controller honors the reservation without extra IAM wiring (unlike Azure's `Network Contributor` dance).

**Files:** `terraform/gcp/traefik_lb.tf` (create)

- [ ] **Step 1: Write `traefik_lb.tf`**

Create `terraform/gcp/traefik_lb.tf`:

```hcl
# Static regional IP for Traefik's LoadBalancer Service. PREMIUM network tier
# (default for new projects) — matches the GKE zonal LB classic Network LB.
# prevent_destroy guards against accidental rotation; the DNS A records in
# dns.tf depend on this IP being stable across cluster rebuilds.
resource "google_compute_address" "traefik" {
  name         = "traefik-${local.name_prefix}"
  region       = var.region
  address_type = "EXTERNAL"
  network_tier = "PREMIUM"

  description = "Static LB IP for Traefik — pinned via Service.spec.loadBalancerIP (TRA-461)"

  labels = local.common_labels

  lifecycle {
    prevent_destroy = true
  }
}
```

- [ ] **Step 1: Edit `clusterissuer.yaml`**

Open `helm/cert-manager-config/templates/clusterissuer.yaml`. Insert a new `{{- else if eq .Values.solver "cloudDNS" }}` branch **between** the existing `azureDNS` branch and the `{{- else }}{{- fail ... }}` block, and update the fail message:

```yaml
      {{- else if eq .Values.solver "azureDNS" }}
      # Workload-identity auth: only managedIdentity.clientID in the auth section.
      # cert-manager's webhook rejects the schema if top-level tenantID/clientID/
      # clientSecretSecretRef appear alongside managedIdentity.
      - dns01:
          azureDNS:
            hostedZoneName: {{ .Values.azureDNS.hostedZoneName }}
            resourceGroupName: {{ .Values.azureDNS.resourceGroupName }}
            subscriptionID: {{ .Values.azureDNS.subscriptionID }}
            managedIdentity:
              clientID: {{ .Values.azureDNS.managedIdentity.clientID }}
        selector:
          dnsZones:
            - {{ .Values.azureDNS.hostedZoneName }}
      {{- else if eq .Values.solver "cloudDNS" }}
      # GKE Workload Identity — no token/secret in spec; cert-manager pod SA
      # is annotated with iam.gke.io/gcp-service-account (set by
      # argocd/root/templates/cert-manager.yaml).
      # NOTE: cloudDNS.hostedZoneName is the Cloud DNS managed-zone RESOURCE
      # NAME (e.g. "gke-trakrf-app"), NOT the DNS name. cloudDNS.dnsZoneName
      # is the DNS name used in the selector.
      - dns01:
          cloudDNS:
            project: {{ .Values.cloudDNS.project }}
            hostedZoneName: {{ .Values.cloudDNS.hostedZoneName }}
        selector:
          dnsZones:
            - {{ .Values.cloudDNS.dnsZoneName }}
      {{- else }}
      {{- fail (printf "Unknown solver type: %q (expected 'cloudflare', 'azureDNS', or 'cloudDNS')" .Values.solver) }}
      {{- end }}

red() { printf "\033[31m❌ %s\033[0m\n" "$*"; }
green() { printf "\033[32m✅ %s\033[0m\n" "$*"; }
yellow() { printf "\033[33m⚠️  %s\033[0m\n" "$*"; }

echo "=== ArgoCD Applications ==="
if ! kubectl -n argocd get applications &>/dev/null; then

CERT_READY=$(kubectl -n traefik get certificate -o json | \
  jq -r '.items[] | select(.status.conditions[]?.type=="Ready" and .status.conditions[]?.status=="True") | .metadata.name' | head -1)

echo "=== Traefik LB IP matches tofu output ==="
TF_IP=$(tofu -chdir="$TF_DIR" output -raw traefik_lb_ip)
SVC_IP=$(kubectl -n traefik get svc traefik -o jsonpath='{.status.loadBalancer.ingress[0].ip}')
if [[ "$TF_IP" == "$SVC_IP" ]]; then
  green "Traefik LB IP $SVC_IP matches tofu reservation"
else
  fail "Traefik LB IP $SVC_IP != tofu $TF_IP"
fi

echo
echo "=== DNS ==="
for name in "$HOST" "foo.$HOST" "$GRAFANA_HOST"; do
  DNS_IP=$(dig +short "$name" | tail -1)
  if [[ "$DNS_IP" == "$TF_IP" ]]; then
    green "$name -> $DNS_IP"
  else
    fail "$name -> $DNS_IP (expected $TF_IP)"
  fi
done

echo
echo "=== Backend HTTP ==="
if curl -sI "https://$HOST" --max-time 10 | head -1 | grep -q '200'; then
  green "https://$HOST returns 200"
else
  fail "https://$HOST did not return 200"
fi

echo
if [[ $FAILURES -eq 0 ]]; then
  green "ALL PRECONDITIONS PASS — proceed to manual UI walkthrough"
  exit 0
else
  red "$FAILURES check(s) failed"
  exit 1
fi
```
