# TRA-459 GKE Phase 1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up `terraform/gcp/` scaffolding (provider, variables, API enablement) against the existing R2 state backend with a new `gcp.tfstate` key, and enable the 5 required APIs on GCP project `trakrf-494211`. No cluster, no billable resources beyond API enablement (free).

**Architecture:** Mirror the `terraform/azure/` phase-1 module shape (`provider.tf`, `variables.tf`, `main.tf`, `outputs.tf` + an `apis.tf` for `google_project_service` entries). State lives in the existing Cloudflare R2 `tf-state` bucket under key `gcp.tfstate`. Project `trakrf-494211` stays click-ops (managed outside Terraform, matching the AKS precedent of not TF-managing the deployment boundary itself).

**Tech Stack:** OpenTofu 1.x, `hashicorp/google` ~> 6.0, `hashicorp/random` ~> 3.6, Cloudflare R2 via `backend "s3"`, justfile for local workflow, GitHub Actions for `tofu validate`, ADC (`gcloud auth application-default login`) for local auth.

**Branch:** `feature/tra-459-gke-phase1-scaffolding` (already created, spec already committed as b9f739e).

**Spec:** `docs/superpowers/specs/2026-04-23-tra-459-gke-phase-1-design.md`.

### Task 2: Write the Terraform module

**Files:**
- Create: `terraform/gcp/provider.tf`
- Create: `terraform/gcp/variables.tf`
- Create: `terraform/gcp/main.tf`
- Create: `terraform/gcp/apis.tf`
- Create: `terraform/gcp/outputs.tf`

These five files are interdependent (locals referenced across them, variables used everywhere) and individually short. Land them together, then validate with `tofu init -backend=false` + `tofu validate` in a later task.

- [ ] **Step 1: Write `terraform/gcp/provider.tf`**

```hcl
terraform {
  required_version = ">= 1.0.0"

  required_providers {
    google = {
      source  = "hashicorp/google"
      version = "~> 6.0"
    }
    random = {
      source  = "hashicorp/random"
      version = "~> 3.6"
    }
  }

  backend "s3" {
    # endpoints injected via backend.conf (generated by justfile from env vars)
    bucket = "tf-state"
    key    = "gcp.tfstate"
    region = "auto"

    skip_credentials_validation = true
    skip_region_validation      = true
    skip_metadata_api_check     = true
    use_path_style              = true
  }
}

provider "google" {
  project = var.project_id
  region  = var.region
  zone    = var.zone
}
```

Auth note: `google` resolves credentials via ADC — no key material committed. The backend "s3" shape is identical to `terraform/azure/provider.tf`; only the `key` and provider blocks differ.

- [ ] **Step 2: Write `terraform/gcp/variables.tf`**

```hcl
variable "project_id" {
  type        = string
  description = "GCP project ID — injected via TF_VAR_project_id in .env.local"
}

variable "region" {
  type        = string
  description = "GCP region for regional resources"
  default     = "us-central1"
}

variable "zone" {
  type        = string
  description = "GCP zone for zonal resources (phase-2 single-zone pin for CNPG PV affinity, TRA-364 pattern)"
  default     = "us-central1-a"
}

variable "environment" {
  type        = string
  description = "Environment name for labels"
  default     = "demo"
}

variable "project" {
  type        = string
  description = "Project label value (not the GCP project — see project_id)"
  default     = "trakrf"
}

variable "location" {
  type        = string
  description = "Region short code used in resource names (e.g. usc1 for us-central1)"
  default     = "usc1"
}

variable "cluster_name" {
  type        = string
  description = "Name of the GKE cluster (applied in phase 2)"
  default     = "trakrf-demo"
}
```

- [ ] **Step 3: Write `terraform/gcp/main.tf`**

```hcl
locals {
  region      = var.region
  name_prefix = "${var.project}-${var.environment}-${var.location}"

  common_labels = {
    project     = var.project
    environment = var.environment
    managed_by  = "terraform"
    ticket      = "tra-459"
  }
}

resource "random_string" "suffix" {
  length  = 4
  special = false
  upper   = false
}
```

Note: GCP label keys/values must be lowercase alphanumeric + `_`/`-`, so `common_labels` differs in case from Azure's `common_tags`. This is intentional, not a typo.

- [ ] **Step 4: Write `terraform/gcp/apis.tf`**

```hcl
# TRA-459: required service APIs for the GKE rollout.
# compute.googleapis.com is a hard transitive dep of GKE (VMs, networks, disks);
# enabling it here avoids a phase-2 apply surprise. Artifact Registry, IAM
# Credentials, and other APIs land alongside the resources that need them in
# TRA-460 and later phases.

resource "google_project_service" "required" {
  for_each = toset([
    "container.googleapis.com",
    "compute.googleapis.com",
    "dns.googleapis.com",
    "iam.googleapis.com",
    "cloudresourcemanager.googleapis.com",
  ])

  service = each.key

  # Keep services enabled if this resource is removed from state.
  # Disabling APIs on a live project can break running resources.
  disable_on_destroy = false
}
```

- [ ] **Step 5: Write `terraform/gcp/outputs.tf`**

```hcl
output "project_id" {
  description = "GCP project ID"
  value       = var.project_id
}

output "region" {
  description = "GCP region"
  value       = var.region
}
```

Outputs are minimal by design — this module has no downstream consumers yet. TRA-460 will add cluster, DNS, and registry outputs.

- [ ] **Step 3: If validate fails**

Read the error. Common causes and fixes:
- Unknown variable: check spelling in `provider.tf`/`apis.tf`/`outputs.tf` against `variables.tf`.
- Unknown attribute on `google_project_service`: the resource only supports `project`, `service`, `disable_on_destroy`, `disable_dependent_services`. Remove anything else.
- Provider schema mismatch: pin `~> 6.0` — newer major versions may have breaking changes.

Fix inline and re-run Step 2 until it passes.
